# A1 Expert Roster

> **System**: ChronicleCore A1 Expert System
> **Active Agents**: 38
> **Architecture**: Human-in-the-Loop, structured persona governance
> **As of**: April 2026

This roster documents all 38 active expert agents in the ChronicleCore A1 system. Each agent operates under a defined identity module comprising a core constraint definition (`SKILL.md`), persistent sovereign memory (`sovereign/`), modular capability packs (`evolutions/`), and an operational protocol library (`references/`).

Agents are classified under the **5 Pillars of Governance** (public topology). For a complete case study of one agent's identity module, see [`examples/inquisitor/`](examples/inquisitor/).

> **Referenced in**: Lee, M.-H. (2026). *Agentic Social Affordance Framework (ASAF): Agent Identity Design as a Collaboration Interface in Multi-Agent Systems.* Frontiers in Computer Science.

## Identity Module Architecture

Each agent in the A1 system is defined by a multi-layer identity module:

```
<agent-id>/
├── SKILL.md                # Core identity constraint definition (required entrypoint)
├── assets/
│   └── persona.md          # Extended character and speaking style specifications
├── sovereign/
│   ├── diary.md            # Longitudinal operational log
│   ├── preferences.md      # Crystallized persona rules (high-weight)
│   └── DIRECTIVES.md       # Standing orders from the Sovereign
├── references/             # Operational protocols and review standards
└── evolutions/             # Modular capability packs (DLCs)
```

This architecture extends the `SKILL.md` pattern (Anthropic, 2025) into a persistent identity framework designed to preserve Social Affordance signal fidelity across sessions — the operational rationale underlying **Memory Crystallization** (separating identity-defining constraints from transient reasoning logs).

**Personality Variance Audit** continuously monitors cross-agent epistemic and rhetorical convergence at runtime, detecting when agents intended to maintain distinct social identities begin producing indistinguishable outputs. This complements the design-time **Personality Uniqueness Check** that prevents redundant social affordance signals when new agents are introduced.
